함수형 프로그래밍은 프로그래밍 패러다임 중 하나로, **순수 함수**와 **불변성**을 강조하는 방식입니다. 어떻게(how)보다는 **무엇(what)에 집중하며, 순수 함수를 조합하는 형태로 코드를 작성한다는 특징**이 있습니다.

먼저, 함수형 프로그래밍의 핵심 개념 중 하나는 **순수 함수** 입니다. 순수 함수란 **동일한 입력에 대해 항상 동일한 결과를 반환하며, 외부 상태를 변경하지 않는 함수**를 의미합니다.
```js
function add(a, b) {
  return a + b;
}
```

이 함수는 인자 a와 b에 따라 항상 같은 값을 반환하며, 외부 상태를 변경하지 않기 때문에 순수 함수입니다. 만약 함수 내에서 데이터베이스에 접근하거나 전역 변수를 변경하는 등 부수 효과를 포함한다면 순수 함수가 아닙니다.

다음으로 중요한 개념은 **불변성** 입니다. 함수형 프로그래밍에서는 데이터를 직접 변경하는 것을 피하고, 기존 데이터를 기반으로 새로운 데이터를 생성하는 방식을 사용합니다. 예를 들어, 배열의 값을 변경할 때 `push` 대신 `concat`을 사용하는 것이 함수형 프로그래밍에 적합한 방식입니다.

```js
const numbers = [1, 2, 3];
const newNumbers = numbers.concat(4);
console.log(newNumbers); // [1, 2, 3, 4]
console.log(numbers); // [1, 2, 3] (원본 배열이 변경되지 않음)
```

이외에도 고차 함수, 커링, 함수 합성, 재귀 등의 기법을 적극적으로 활용하기도 합니다.

## 이런 식으로 코드를 짤 때 어떤 장점이 있나요? 🤔
여러 장점이 있지만, 크게 세 가지 장점에 대해 말씀드리겠습니다.

첫째, **코드의 유지보수성이 좋습니다.** 함수형 코드는 절차적이지 않고 선언적이기 때문에 코드가 읽고 수정하기 좋은 형태입니다.

둘째, **테스트하기 용이합니다.** 순수 함수는 외부 의존성이 없기 때문에 테스트하기 쉬운 형태이기 때문입니다.

셋째, **병렬 처리하기 용이합니다.** 불변성 및 부수 효과가 없다는 특징 덕분에 병렬 처리하기에 적합합니다.
